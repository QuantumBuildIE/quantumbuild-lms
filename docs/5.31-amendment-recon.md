## 2. Recommendations

### Gate expression (§2)

```csharp
if (environment.IsDevelopment() || environment.IsEnvironment("Demo"))
```

`IHostEnvironment.IsEnvironment(string)` is case-insensitive (`StringComparison.OrdinalIgnoreCase`), so `ASPNETCORE_ENVIRONMENT=Demo`, `demo`, or `DEMO` all match. Keep the inline disjunction — two values don't warrant a helper.

### Test update (§3)

**Option A — keep the existing two tests only.**

"Testing" is neither "Development" nor "Demo"; both tests continue to pass without modification. The Demo-includes-Demo inclusion behaviour is readable directly from the one-line change. The warn-and-skip under missing Seed config is already implicitly covered (Testing environment also has no Seed config and doesn't seed). Adding a Demo-environment test variant would require a new `CustomWebApplicationFactory` subclass with `UseEnvironment("Demo")` plus in-memory Seed config — infrastructure overhead for obvious coverage.

### Note 31 amendment (§4)

Replace the title line and Bootstrap paragraph (CLAUDE.md:1426 and 1435):

**Title line** (line 1426):
> **Note 31 — DataSeeder credentials are Development and Demo-sourced from config**: `DataSeeder.SeedAsync` creates two credentialled accounts only when `environment.IsDevelopment() || environment.IsEnvironment("Demo")` is true. Credentials are read from configuration:

**Bootstrap paragraph** (line 1435) — replace with:

> **Bootstrap pattern by environment:**
>
> - **Demo:** Seeded automatically on first deploy provided `Seed__SuperUser__Email`, `Seed__SuperUser__Password`, `Seed__Admin__Email`, and `Seed__Admin__Password` are set as Railway env vars (ASP.NET Core env-var separator is `__` not `:`). Missing values log a warning and skip that account — no throw.
> - **Production:** SuperUser must still be inserted via direct DB script or pre-deploy migration (per Note 20). The seeder does not run for Production.

## 3. Edit (1) — RefresherFrequencyMapper (new file)

**File:** `src/Modules/ToolboxTalks/QuantumBuild.Modules.ToolboxTalks.Domain/Helpers/RefresherFrequencyMapper.cs`

New static class in Domain/Helpers that owns the mapping in both directions. All four write paths import this class — the duplicated inline switch logic from the recon's adjacent observation 9.3 is eliminated.

Three methods:

- **`ToLegacyFrequency(bool requiresRefresher, int intervalMonths) → ToolboxTalkFrequency`** — canonical → legacy. Maps Once (`!requiresRefresher`) to `Once`; 1 month to `Monthly`; 2–3 months (Quarterly) to `Monthly`; ≥12 months to `Annually`; mid-range (4–11 months) to `Monthly` (closest legacy bucket).

- **`ToCanonicalFields(ToolboxTalkFrequency frequency, int existingIntervalMonths = 12) → (bool, int)`** — legacy → canonical. `Once` and `Weekly` → `(false, existingIntervalMonths)` (Weekly has no months equivalent and was never functional in `RefresherSchedulingService`; existing interval is preserved for clean round-trip if the user later selects a non-Once value). `Monthly` → `(true, 1)`. `Annually` → `(true, 12)`.

- **`FromWizardFrequencyString(string? wizardFrequency, int existingIntervalMonths = 12) → (bool, int)`** — wizard string → canonical. `"Monthly"` → `(true, 1)`, `"Quarterly"` → `(true, 3)`, `"Annually"` → `(true, 12)`, `"Once"` or null/unrecognised → `(false, existingIntervalMonths)`. The `existingIntervalMonths` parameter preserves the current interval when Once is selected in Step 4, matching the original handler's behaviour.

## 4. Edit (2) — InitialiseToolboxTalkCommandHandler

**File:** `src/Modules/ToolboxTalks/QuantumBuild.Modules.ToolboxTalks.Application/Commands/InitialiseToolboxTalk/InitialiseToolboxTalkCommandHandler.cs`

Three changes:

**a) Replace inline switch with mapper call:**
```csharp
// Before
var (defaultRequiresRefresher, defaultRefresherIntervalMonths) = refresherFrequency switch
{
    "Monthly" => (true, 1),
    "Quarterly" => (true, 3),
    "Annually" => (true, 12),
    _ => (false, 12),
};

// After
var (defaultRequiresRefresher, defaultRefresherIntervalMonths) =
    RefresherFrequencyMapper.FromWizardFrequencyString(refresherFrequency);
```

**b) Add `Frequency` to ToolboxTalk initializer:**
```csharp
RequiresRefresher = defaultRequiresRefresher,
RefresherIntervalMonths = defaultRefresherIntervalMonths,
Frequency = RefresherFrequencyMapper.ToLegacyFrequency(defaultRequiresRefresher, defaultRefresherIntervalMonths),
```

New-wizard talks now have a correct `Frequency` from creation. Dashboard frequency breakdown and admin list filter produce accurate results immediately.

**c) Fix hardcoded `FrequencyDisplay = "Once"` in MapToDto (recon observation 9.1):**
```csharp
// Before
FrequencyDisplay = "Once",

// After
FrequencyDisplay = entity.Frequency switch
{
    ToolboxTalkFrequency.Once => "One-time",
    ToolboxTalkFrequency.Weekly => "Weekly",
    ToolboxTalkFrequency.Monthly => "Monthly",
    ToolboxTalkFrequency.Annually => "Annually",
    _ => entity.Frequency.ToString()
},
```

## 5. Edit (3) — UpdateToolboxTalkSettingsCommandHandler (new wizard Step 4)

**File:** `src/Modules/ToolboxTalks/QuantumBuild.Modules.ToolboxTalks.Application/Commands/UpdateToolboxTalkSettings/UpdateToolboxTalkSettingsCommandHandler.cs`

```csharp
// Before
(talk.RequiresRefresher, talk.RefresherIntervalMonths) = request.RefresherFrequency switch
{
    RefresherFrequency.Monthly => (true, 1),
    RefresherFrequency.Quarterly => (true, 3),
    RefresherFrequency.Annually => (true, 12),
    _ => (false, talk.RefresherIntervalMonths)
};

// After
(talk.RequiresRefresher, talk.RefresherIntervalMonths) =
    RefresherFrequencyMapper.FromWizardFrequencyString(
        request.RefresherFrequency.ToString(), talk.RefresherIntervalMonths);

talk.Frequency = RefresherFrequencyMapper.ToLegacyFrequency(talk.RequiresRefresher, talk.RefresherIntervalMonths);
```

`request.RefresherFrequency.ToString()` maps the wizard enum values (`Once/Monthly/Quarterly/Annually`) to the same strings the mapper expects. The existing interval is preserved when Once is selected. The Frequency mirror write means the handler's returned DTO now reflects the correct `Frequency` value (previously the DTO read stale `Frequency` after Step 4 updated only the new fields).

## 6. Edit (4) — UpdateToolboxTalkCommandHandler (old wizard edit — the critical overwrite fix)

**File:** `src/Modules/ToolboxTalks/QuantumBuild.Modules.ToolboxTalks.Application/Commands/UpdateToolboxTalk/UpdateToolboxTalkCommandHandler.cs`

```csharp
// Before — old form omits requiresRefresher/refresherIntervalMonths; DTO defaults
// (false/12) were silently written, overwriting Step 4 values
toolboxTalk.RequiresRefresher = request.RequiresRefresher;
toolboxTalk.RefresherIntervalMonths = request.RefresherIntervalMonths;

// After — derive from Frequency (the old wizard's canonical field); DTO defaults ignored

    RefresherFrequencyMapper.ToCanonicalFields(request.Frequency, toolboxTalk.RefresherIntervalMonths);
```

`request.RequiresRefresher` and `request.RefresherIntervalMonths` are now ignored. `Frequency` is the old wizard's single write point for refresher intent. The `existingIntervalMonths` parameter preserves the current `RefresherIntervalMonths` value when `Frequency = Once` or `Weekly`, so a subsequent Step 4 edit doesn't lose the configured interval.

`toolboxTalk.Frequency = request.Frequency` at line 118 is unchanged — this is the correct write.

## 8. Edit (6) — ToolboxTalkForm.tsx (frontend Weekly removal)

**File:** `web/src/features/toolbox-talks/components/ToolboxTalkForm.tsx`

Approach 2 from the fix spec: filter at point of use in the form, not in the constant. `Weekly` is excluded for new selections but shown for existing talks that currently have `Frequency = Weekly`.

```tsx
// Before
{FREQUENCY_OPTIONS.map((option) => (
  <SelectItem key={option.value} value={option.value}>
    {option.label}
  </SelectItem>
))}

// After
{FREQUENCY_OPTIONS.filter(
  (opt) => opt.value !== 'Weekly' || talk?.frequency === 'Weekly'
).map((option) => (
  <SelectItem key={option.value} value={option.value}>
    {option.label}
  </SelectItem>
))}
```

`web/src/lib/constants/frequency.ts` is unchanged — `FREQUENCY_OPTIONS` and `FREQUENCY_VALUES` retain `Weekly` for display lookups.

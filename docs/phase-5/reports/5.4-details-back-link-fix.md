# 5.4 — Details Back-Link Fix

**Date:** 2026-06-12  
**Branch:** transval  

---

## ROOT CAUSE

`ValidateStep.tsx:179` linked to the validation detail page with no context marker, and `validation/[runId]/page.tsx:17` had `onBack` hardcoded to the talk's standalone page. Clicking Details from the wizard then Back ejected the user from the wizard.

---

## IN-SCOPE WORK

**A. ValidateStep.tsx — append `?from=wizard`**

- File: `web/src/features/toolbox-talks/components/learning-wizard/steps/ValidateStep.tsx`
- Line 179: changed href from  
  `` `/admin/toolbox-talks/talks/${talkId}/validation/${run.id}` ``  
  to  
  `` `/admin/toolbox-talks/talks/${talkId}/validation/${run.id}?from=wizard` ``

**B. Validation detail page — branch `onBack` by query param**

- File: `web/src/app/(authenticated)/admin/toolbox-talks/talks/[id]/validation/[runId]/page.tsx`
- Line 3: added `useSearchParams` to import
- Lines 9–11: read `searchParams.get('from') === 'wizard'` into `fromWizard`
- Lines 16–22: `onBack` branches to `/admin/toolbox-talks/learnings/${talkId}/validate` when `fromWizard`, otherwise falls back to the talk's standalone page (preserving all other entry-point behaviour)

No changes to `ValidationRunDetailView` — `onBack` is already a consumed prop.
